Session 1: Real-World Application Test (ReqRes API)

This session used the ReqRes API (https://reqres.in/) to test basic CRUD operations.

Test 1: Reading Data (GET)

Scenario: You are working on a web application and need to fetch the data for a single user whose ID is 2 to display on their profile page.

Your Attempt: 
GET https://reqres.in/useriD=1
Correct Answer & Explanation:

Method: GET
URL: https://reqres.in/api/users/2

The GET method was correct. The URL needed correction: for a single resource, the ID (2) should be part of the path. The path was also missing the /api/ segment. Query parameters (?) are typically for filtering lists, not fetching single items by ID.

Test 2: Creating Data (POST)

Scenario: A new user, 'Morpheus', has just signed up. His job title is 'leader'. You need to create a new record for him in the system.

Your Attempt: 
methd = get, url is regres.in/api/users and json body is
Correct Answer & Explanation:

Method: POST
URL: https://reqres.in/api/users

Body (Raw JSON):

{
    "name": "morpheus",
    "job": "leader"
}

The method to create new data is POST. The URL had a typo (regres instead of reqres). A POST request also requires a JSON body containing the data for the new user.

Test 3: Updating Data (PUT)

Scenario: Morpheus has been promoted! You need to update his record (user ID 2) to change his job title to 'zion resident'.

Your Attempt: 
POST https://regres.in/api/users/2

Body:
{
  "Name": "Morpheus",
  "Job": "zion rezident",
  "userId": 2
}
Correct Answer & Explanation:

Method: PUT
URL: https://reqres.in/api/users/2

Body (Raw JSON):

{
    "name": "morpheus",
    "job": "zion resident"
}

The method to update existing data is PUT. The URL had a typo. The JSON body should use lowercase keys (name, job) as expected by the API, and the redundant userId is not needed.

Test 4: Deleting Data (DELETE)

Session 2: Bolt Food API Practice

This session uses a fictional API for the Bolt Food app. Assumed base URL is https://api.boltfood.com/v1.

Practice 1: Filtering a List of Restaurants

Scenario: You've just opened the app. You want to find all the restaurants that serve sushi.

Your Attempt: 
GET https://api.boltfood.com/v1
Correct Answer & Explanation:

Method: GET
URL: https://api.boltfood.com/v1/restaurants?cuisine=sushi

The GET method was correct. The URL needed two additions: the /restaurants resource path to specify what you're looking for, and the ?cuisine=sushi query parameter to filter the list.

Phase 3: Automation & Advanced Testing
